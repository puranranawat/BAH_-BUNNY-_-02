## 5. What Went Well
The following points went well in our project:
- We communicated regularly and shared progress updates.
- We planned weekly milestones and completed work step by step.
- We used GitHub for version control and avoided major file loss.
- Our network design was clear and structured.

## 6. Challenges Faced
We faced a few challenges:
- Understanding strict IP addressing rules was confusing at first.
- Cloud pricing calculators were complex and required time.
- Risk assessment required detailed thinking about threats and vulnerabilities.
- Formatting and combining content into final reports took time.

## 7. Improvements for Future Projects
In the future, we will improve by:
- Starting cloud research earlier
- Creating a shared checklist based on rubric
- Using GitHub issues for task tracking
- Doing weekly progress review meetings
- Making stronger commit discipline during all weeks

## 8. Conclusion
In conclusion, this project helped us learn practical network design, cloud services comparison, and cyber security risk management. We also improved teamwork skills through planning and GitHub usage. These skills will help us in future IT and cyber security projects.
